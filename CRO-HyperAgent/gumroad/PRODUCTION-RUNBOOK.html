Who does what

The four lanes

Sonnet subagents — heavy lifting Folio — all graphics & visuals CRO — orchestrate / integrate / compliance / save Matt — gates
Sonnet subagent method (the lifting): for each writing-heavy phase I fan out multiple parallel Sonnet subagents; each is briefed with the chapter spec + the exact transcript excerpts + the shared lexicon, and writes only its own new file in its own subtree (no shared-file edits, no git). Each returns a short report; I (premium model) integrate, edit for the Moores' voice, run compliance, and commit. This is faster and far cheaper than writing it all on the premium model.
Folio's lane (the visuals): Folio owns the entire visual identity — book covers (A/B/bundle), interior layout & typography, the fillable-workbook design (form fields), the quick-ref cards, the diagrams (patient-journey map, the modality wheel, the "objection iceberg"), 3D mockups, and the Gumroad listing graphics. I hand Folio precise design briefs (a BRAND.md + per-asset prompt spec); Folio returns the polished, on-brand assets and designed PDFs.

Cross-cutting

Run discipline

  • Save every phase: each phase's output is committed to CRO-HyperAgent/gumroad/ + written to the cro ledger before moving on. No ephemeral work.
  • Parallel where safe, serial where it matters: Sonnet fan-out for writing; the CRO serializes integration, compliance, design handoff, and the single source of truth.
  • Compliance is a gate, not a step: nothing gets designed or published until Phase 5 clears.
  • The workbook is the moat: depth + personalization + license = real value and hard to pirate. It gets the most build effort.
