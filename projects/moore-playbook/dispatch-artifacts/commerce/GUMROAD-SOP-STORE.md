## 4. Store setup steps (the ~60 minute Matt path, copy-paste first)

Gumroad has no public write API for product creation that we operate today, so creation is a short manual path. Steps are ordered so the store is sellable in one sitting.

1. **Create the account.** Go to gumroad.com, sign up as a creator with the OTCHealth business email. Set the store name to OTC Retail Compliance Lab and the handle to `otcretailcompliancelab`. Paste the store description and tagline from Section 1.
2. **Connect the payout rail.** Settings, Payments. Add the bank account or PayPal that receives payouts. This is the only money-out gate and it is one time. (Note: this is a separate rail from the TReO Stripe payout bank. They do not block each other.)
3. **Add the 4 single products.** For each of Listings 1 to 4: New Product, type Digital Product, paste the title, price, permalink slug, summary, and description from Section 2. Upload its editable DOCX files plus the PDF reference copy. Confirm instant delivery is ON.
4. **Add the bundle.** Create Listing 5 as its own product at $149 and upload all four kits into it. (Gumroad also supports a native Bundle product type that points at existing products. Either approach works. The single-product-with-all-files approach is simplest and delivers instantly.)
5. **Set the receipt and refund text.** Paste the store-wide receipt note and refund policy from Section 1 into the relevant Gumroad settings.
6. **Publish all 5.** Flip each from draft to published.
7. **Run one test purchase.** Buy the $59 listing with a real card (or a Gumroad test path), confirm the file auto-delivers to the buyer email and appears in the library, then refund that single test if a real card was used. This proves the delivery loop end to end.
8. **Hand back the live URLs.** Report the five live product URLs so the content engine and the warm-list email can link them.
