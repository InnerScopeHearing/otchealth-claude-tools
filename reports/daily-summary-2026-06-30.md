### token-keeper (`skills/token-keeper/`)

New skill wired to the nightly Azure Container App Job for OAuth refresh-token rotation.

**What broke:** the QuickBooks provider was reading placeholder secret names
(`quickbooks-client-id`, `quickbooks-refresh-token`) that don't exist. Every
nightly run silently no-op'd (`MISSING_SECRETS`), masked by `|| true` in the job
shell, so `lastRefresh` was always `null`.

**What was fixed:**
- Corrected secret names to `qbo-client-id` / `qbo-client-secret`.
- QBO has **4 separate rotating refresh tokens** — one per entity:
  `qbo-refresh-otchealth`, `qbo-refresh-innd`, `qbo-refresh-hearingassist`, `qbo-refresh-personal`.
- Introduced `refreshOneOAuth()` / `refreshOAuth()` multi-tenant loop; Xero stays
  single-tenant (byte-identical code path).
- `status` and `create-slots` commands now enumerate all per-entity secrets.
- Verified by dry-run: all 4 entities resolve correctly.

**Files:** `skills/token-keeper/keeper.mjs` (297 lines), `SKILL.md`, `run.sh`

### ocr-sweep (`skills/ocr-sweep/`)

Azure Document Intelligence auto-OCR sweep for the legal + CFO document stores.

**What broke:** `sweep.mjs` buffered each blob in memory using `CONC=6` concurrent
workers. One oversized file OOM-killed the 1 Gi container; the
`docintel-ocr-sweep` job had been failing for 30+ hours before this fix.

**What was fixed:**
- Added `MAX_MB` guard (default 200 MB). Checks `Content-Length` **before**
  `arrayBuffer()` is called, so an oversized blob is skipped without loading it
  into memory.
- Job resource limits bumped to **4 Gi RAM / 2 CPU** and `CONC` dropped to 2
  (applied via ARM template, not in this file).

**File:** `skills/ocr-sweep/sweep.mjs` (69 lines, new)
